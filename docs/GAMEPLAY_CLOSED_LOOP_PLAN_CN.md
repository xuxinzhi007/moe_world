# 玩法闭环修复计划

## 目标

把当前半成品收敛成一条可完整跑通的 MVP 闭环：

`大厅 -> 大世界接主线 -> 完成击杀/收集 -> 领取奖励 -> 进入试炼 -> 试炼通关/失败结算 -> 返回大世界/大厅`

## 当前缺点

- 目标不够明确，玩家进入世界后不知道当前要优先做什么
- 主线任务有雏形，但任务完成后与试炼入口的关系不够清晰
- 试炼之前是无限波次，没有明确“通关点”
- 顶部提示更多是功能说明，不是当前目标说明
- 奖励和终点之间缺少“完成一局就算闭环”的结算感
- 美术素材风格不统一，但先补玩法闭环更重要

## 这版 MVP 的终点定义

玩家完成以下步骤即视为闭环完成：

1. 在大厅进入大世界
2. 找向导露露接取任务
3. 击败 3 只怪物并收集 3 份史莱姆凝胶
4. 返回向导露露领取奖励
5. 进入试炼并打完固定目标波次
6. 看到结算面板并返回大世界

## 当前执行顺序

### P0

- 任务状态公开给 UI
- 大世界顶部提示改成“当前主目标”
- 试炼加入明确通关波次
- 试炼传送门在主线未完成时提示“先完成向导任务”

当前状态：
- 已完成：任务状态已通过 `QuestManager` 对外提供，大世界顶部提示已切换为主目标文本
- 已完成：试炼已改为固定目标波次，并在清场后结束，不再无限持续
- 已完成：试炼传送门在主线未解锁时会明确提示“先完成向导任务”

### P1

- 补齐试炼结算后的奖励回流说明
- 补齐大厅返回后的“本轮完成”反馈
- 抽离世界场景里剩余的控制器

当前状态：
- 已完成：试炼结算已区分“通关 / 失败 / 中途离场”，只有通关才结算主线完成与奖励回流
- 已完成：返回大厅后会显示“本轮完成”提示，大厅单机入口文案会同步当前闭环进度
- 进行中：世界场景控制器仍待继续拆分，当前只完成了静态配置抽离

### P2

- 在闭环稳定后再统一替换少量素材
- 优先替换最显眼的 NPC / 怪物 / 大件装饰

## 验收标准

- 玩家从大厅能自然知道第一目标
- 任务完成后能自然知道下一目标是试炼
- 试炼不会无限持续，存在明确结束点
- 结算后能回到世界或大厅，流程不阻塞
- 整体体验是“一个完整小回合”，而不是一堆散功能
