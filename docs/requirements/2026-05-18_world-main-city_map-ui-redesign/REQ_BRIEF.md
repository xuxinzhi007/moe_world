# 需求简报

## 文档元信息

- 需求包目录：`docs/requirements/2026-05-18_world-main-city_map-ui-redesign/`
- 日期：2026-05-18
- 提出者：用户
- 当前状态：in_progress

## 1. 需求名称

- 主城地图与 UI 收口重构

## 2. 目标

- 明确 `WorldScene/world_main` 是玩家出生地图与主城中心。
- 修复地图切换后主城无法正确往返、主城可走出边界、主城被显示为“未知区域”等问题。
- 将边缘切换交互改成与 `NPC` / `survivor_portal` 相同的交互模型。
- 将右上角地图改为真实当前区域 minimap，而不是只显示实体雷达点。

## 3. 使用场景

- 玩家出生在主城，进行接任务、补给、交互、准备。
- 玩家走到主城边缘时，看到前往周边区域的提示，并主动确认切换。
- 玩家在周边区域探索、刷怪、采集后，可以明确返回主城。
- 玩家通过右上角小地图快速理解当前地图边界、出口、自己位置和朝向。

## 4. 范围

- 做：
- 主城中心化结构修正
- 主城与周边区域的邻接关系修正
- 边缘切换交互重做
- 主城边界拦截与提示
- 真实小地图首版
- 全屏地图文案与主城命名收口

- 不做：
- 整体改做 3D 场景
- 大规模替换现有地图美术资源
- 本轮内完成所有新区域内容填充

## 5. 平台与模式

- 平台：PC + 移动端
- 模式：优先单机大世界，兼顾现有联机逻辑不回退

## 6. 验收标准

- [ ] 玩家出生后顶部区域标签显示为“主城”
- [ ] 主城不能直接走出有效地图边界进入无效区域
- [ ] 主城边缘切换改为提示式交互，玩家确认前不切图
- [ ] 玩家从外部区域可以稳定返回主城
- [ ] 右上角小地图能展示当前地图边界、出口、玩家点位
- [ ] 全屏地图以主城为中心展示相邻区域，不再误指向旧 `plaza`

## 7. 优先级与顺序

- 优先级：P0
- 顺序：
1. 文档定案
2. 交互修正
3. 主城边界修复
4. 小地图首版重构
5. 验证与收口

## 8. 备注

- 当前问题本质是“地图结构职责混用 + UI 交互模型不一致”，不是简单参数错误。
- 后续如果继续扩区，建议始终以“主城 hub + 周边外区”的 2D ARPG 结构扩展。
