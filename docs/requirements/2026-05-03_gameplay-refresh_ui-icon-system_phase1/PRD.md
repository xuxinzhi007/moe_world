# PRD：UI 图标系统完善（阶段一）

## 1. 功能概述

- 功能名称：UI 图标系统完善（全 UI 统一 + 规范落地）
- 面向对象：所有玩家（PC/移动端）
- 核心价值：降低认知成本，提升操作反馈与职业识别度

## 2. 功能流程

- 入口：玩家进入大厅/大世界/试炼时加载对应 UI。
- 主流程：
  1. UI 组件按统一图标规范加载资源；
  2. 不同职业展示对应攻击/技能图标；
  3. 背包、地图、设置等按钮按统一规格展示。
- 失败/中断处理：
  - 图标缺失时使用默认后备图标；
  - 路径失效时输出一次性诊断日志并回退默认图标。
- 退出与回退：
  - 切场景后图标状态保持正常，返回大厅或重进世界不丢失。

## 3. 交互规则

- PC 端：鼠标悬停/点击态图标清晰，尺寸与对齐统一。
- 移动端：触控按钮图标居中且可读，避免被裁切。
- 文案与提示：保持现有中文文案，不新增冗余提示。

## 4. 状态与边界

- 正常状态：图标按业务场景正确显示。
- 异常状态：缺图、路径错误、尺寸异常时自动回退默认资源。
- 边界条件：
  - 不改动按钮逻辑，只替换/统一图标展示系统。
  - 不引入新输入动作映射。

## 5. 数据与配置

- 需要新增/修改：
  - 图标规范文档（命名、尺寸、路径、状态图）；
  - 图标映射配置（职业 -> 攻击图标/技能图标）。
- 存档/持久化影响：无（纯展示层）。

## 6. 验收标准

- [ ] 大世界与试炼 UI 图标统一且可读。
- [ ] PC 与移动端均无明显错位、拉伸、裁切。
- [ ] 职业图标与功能含义一致，回退机制有效。
