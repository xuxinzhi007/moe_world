# 需求简报：玩法单调 / NPC僵硬 / UI图标不完善

## 1. 需求名称

- 玩法体验增强（阶段一：UI 图标系统完善）

## 2. 目标

- 优先解决“UI 图标不完善”问题，提升界面可读性与职业辨识度。
- 为下一阶段的 NPC 生动化（巡逻+驻足）预留规范与扩展位。

## 3. 使用场景

- 玩家在大世界与试炼中进行战斗、背包查看、地图与设置操作时。
- 涉及 PC 与移动端 UI 展示一致性。

## 4. 范围

- 做：
  - 建立统一图标规范（尺寸、命名、目录、状态图）。
  - 完成全 UI 图标统一（战斗 + 背包 + 地图 + 设置）。
  - 补齐缺失图标与错误引用，统一路径到分类目录。
- 不做：
  - 本轮不改核心战斗数值与怪物机制。
  - 本轮不实现 NPC 社交/昼夜作息（仅保留下一阶段方向）。

## 5. 平台与模式

- 平台：PC + 移动端。
- 模式：单机优先，保持现有联机路径不回归。

## 6. 验收标准（必须可验证）

- [ ] 所有核心 UI 按钮均有明确图标，无空白/错位/拉伸异常。
- [ ] 职业相关攻击与技能图标可区分，且在 PC/移动端显示一致。
- [ ] 图标资源路径、命名、尺寸规则有文档沉淀，可复用到后续功能。

## 7. 优先级与时间

- 优先级：P0（先完成图标系统）。
- 实施节奏：文档定稿 -> 小步替换 -> 双端回归。

## 8. 备注

- 你已确认：本轮优先 UI；NPC 方向为“区域巡逻+驻足+随机朝向（低风险）”。
