# TECH DESIGN：UI 图标系统完善（阶段一）

## 1. 技术目标

- 在不改玩法逻辑的前提下，统一 UI 图标资源与显示行为。
- 建立可扩展的图标映射规则，支撑后续职业与技能扩展。

## 2. 影响范围

- 场景：
  - `Scenes/ui/*`
  - `Scenes/SurvivorMobileHud.tscn`（若仍作为 UI 入口容器）
  - `Scenes/MobileControls.tscn`（若仍存在并被引用）
- 脚本：
  - `Scripts/ui/mobile_controls.gd`
  - `Scripts/world/world_scene.gd`（若负责 UI 初始化/刷新）
  - `Scripts/survivor/survivor_arena.gd`（试炼 UI 挂载路径）
- 资源目录：
  - `Assets/ui/`

## 3. 方案设计

- 节点结构改动：
  - 保持原按钮结构，仅统一 icon 赋值方式与尺寸缩放策略。
- 脚本职责改动：
  - UI 控制脚本集中处理 icon 加载、缩放、居中、后备图逻辑。
- 信号/事件流：
  - 保持现有按钮交互信号，不新增并行输入系统。
- 数据读写：
  - 使用常量映射或字典映射职业图标路径；
  - 对缺失资源做安全判断（`get_node_or_null` / `ResourceLoader.exists`）。

## 4. 兼容与风险

- 对现有功能影响：
  - 主要为展示层，行为逻辑不变，风险可控。
- 可能回归点：
  - 不同分辨率下按钮图标错位；
  - 旧路径残留导致部分图标不生效。
- 回滚方案：
  - 保留旧图标映射常量一版，分支对比后可快速回退。

## 5. 实施顺序

1. 统一图标规范文档与路径表。
2. 改 UI 脚本的图标加载与缩放。
3. 分场景回归（大厅/世界/试炼，PC/移动端）。

## 6. 验证方式

- 运行路径：大厅 -> 大世界 -> 试炼 -> 返回大厅。
- 关键日志：缺图回退日志仅打印一次。
- 手测步骤：
  - 切换职业查看攻击/技能图标；
  - 打开背包/地图/设置检查图标一致性；
  - 移动端触控按钮图标检查对齐与点击区域。
