## 下一步推荐结构

1. 先做结构收口，不先做大改版式。
2. 再补玩法闭环缺口，把奖励、消耗、失败结算串起来。
3. 最后做 UI 精修，而不是现在重做整套界面。

### 结构收口建议

- 页面层：`HallScene`、`LoginScreen`、`RegisterScreen`、`ProfileScene` 继续作为独立入口页。
- HUD 层：`WorldGameplayHud` 与试炼 HUD 继续复用同一套交互部件，避免分叉。
- Overlay 层：背包、成长、商店、地图统一成同一类弹层规范，统一打开/关闭/遮罩/标题区。
- Theme 层：以 `Scripts/meta/ui_theme.gd` 为单一主题入口，继续收敛颜色、圆角、按钮、输入框、间距 token。
- 业务层：`world`、`survivor`、`auth` 脚本负责数据和交互，不继续承载过多页面布局重排职责。

## UI 是否需要调整

- 需要调整，但不建议重做。
- 当前问题主要是“结构与规范收口不够”，不是“视觉方向完全错误”。
- 优先调整三类内容：
  - 用统一资源替换大厅里仍在使用的 emoji/临时字符图标。
  - 收敛 HUD 与弹层的响应式布局，减少绝对定位带来的分辨率风险。
  - 统一 Overlay 的标题、关闭按钮、空状态、按钮层级和间距。
