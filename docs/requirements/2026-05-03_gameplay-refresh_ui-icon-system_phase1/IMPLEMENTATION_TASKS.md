# 执行任务单：UI 图标系统完善（阶段一）

## 目标

- 完成全 UI 图标统一，并建立可复用的图标规范。

## 范围

- 做：
  - 战斗、背包、地图、设置相关 UI 图标统一。
  - 职业攻击/技能图标映射统一。
  - 缺失图标后备机制与一次性诊断日志。
- 不做：
  - 不调整战斗数值与怪物 AI。
  - 不实现 NPC 作息与社交行为（留到阶段二）。

## 约束

- 复用现有 UI 场景与脚本，不新增平行 UI 系统。
- 保持中文文案与按钮行为不变。
- 世界/试炼双场景都要验证。
- PC + 移动端都要核对。

## 实施步骤

1. [x] 整理图标资源与命名规范（`Assets/ui/`）。
2. [x] 建立图标映射表（职业/按钮类型 -> 图标路径）。
3. [x] 改 UI 加载逻辑（居中、缩放、回退图）。
4. [ ] 在大世界与试炼分别验证。
5. [ ] 清理旧路径与冗余资源引用。

## 验收标准

- [ ] 全部核心 UI 图标显示正确，无空白。
- [ ] 职业图标可区分且切换正确。
- [ ] PC/移动端均无错位、拉伸、裁切。
- [ ] 缺图时能回退并打印一次诊断日志。

## 风险与回归点

- 旧路径残留导致个别界面图标失效。
- 移动端分辨率差异导致布局偏移。

## 架构约束

- 复用现有场景/autoload，不创建重复系统。

## 闭环要求

- [进入场景] -> [加载图标] -> [交互反馈正确] -> [切场景返回仍正确]
