# 完整需求分阶段执行文档（先文档后开发）

## 1. 目标

在不做大重构的前提下，分阶段完成以下模块闭环：
- 地图与背景
- 角色与动作
- 任务系统
- 材料与图标
- 音频事件
- 怪物遭遇

并保证主流程稳定：大厅 -> 世界 -> 战斗/交互 -> 返回大厅。

## 2. 本轮约束（按你要求）

1) 不并发开启大量子代理。  
2) 每次只做一个小任务并可立即验收。  
3) 每个小任务都要写清：改动文件、验收步骤、回归风险。  
4) 优先 P0 闭环，再扩展 P1/P2 内容。

## 3. P0 执行顺序（当前版本）

### P0-1 任务主链路 MVP（当前先做）
- 目标：实现 1 条完整任务链：接取 -> 推进 -> 完成 -> 领奖。
- 触发：与 `向导露露` 交互接取/领奖。
- 推进条件（MVP）：
  - 击杀怪物达到目标数；
  - 收集指定材料达到目标数。
- 验收：可在世界内完整走通并获得奖励。

### P0-2 怪物掉落 -> 材料拾取 -> 背包显示
- 目标：保证掉落、拾取、背包列表显示一致。
- 验收：怪物掉落后可拾取，背包数量实时变化。

### P0-3 音频事件最小矩阵
- 目标：补齐大厅/世界/战斗/拾取/任务完成的音频触发。
- 验收：关键事件均有音频反馈，且不会失控重叠。

### P0-4 角色动作与怪物遭遇联动收口
- 目标：动作触发、受击反馈、击杀反馈与遭遇循环一致。
- 验收：移动/攻击/技能/击杀循环稳定，主流程无回归。

## 4. 本轮计划改动边界

- 允许改动：`Scripts/autoload/`、`Scripts/world/`、`Scripts/ui/`、`project.godot`。
- 暂不改动：大规模场景结构、核心网络流程、登录注册链路。
