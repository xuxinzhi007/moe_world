# 需求选项确认器

## 目标

把“说不清”变成“可选择”，减少来回沟通成本。

## 何时触发

- 用户同时提出多个目标。
- 需求有明显多种实现路径。
- 用户没有给优先级或验收标准。

## 执行方式

1. 先复述用户目标（1-2句）。
2. 只提取 1-3 个关键分歧点。
3. 每个分歧点给 2-4 个明确选项（短句、可执行）。
4. 用户选定后，输出最终执行版本。

## 输出模板

```markdown
需求理解：
- ...

请确认以下选项：
1) [分歧点A]
- A1 ...
- A2 ...

2) [分歧点B]
- B1 ...
- B2 ...

确认后执行版：
- 目标：
- 范围：
- 步骤：
- 验收：
```

## 约束

- 不一次性抛太多问题（最多 3 个分歧点）。
- 每个选项都必须可落地，不写空话。
