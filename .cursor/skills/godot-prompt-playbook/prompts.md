# Godot 项目提示词模板（可直接复制）

## 1) 页面实现模板（UI）

```text
你是 Godot 4.4 UI 开发工程师。请在现有项目中实现页面改动。

目标：
- [写清要做的页面与效果]

约束：
- 只做最小必要改动，优先复用现有节点与样式函数。
- 保持中文文案不变（除非我明确要求修改）。
- 同时考虑 PC 和移动端布局。

必须对照：
- docs/FLOW_UI_DESIGN_BLUEPRINT.md
- [指定 .tscn]
- [指定 .gd]

验收：
- 给出改动文件列表
- 给出关键交互说明
- 给出手测步骤（PC + 移动端）
```

## 2) 功能开发模板（玩法/系统）

```text
你是 Godot 4.4 玩法程序。请实现以下功能。

功能目标：
- [功能描述]

技术约束：
- 不破坏现有流程（大厅->世界->试炼->返回）
- 若涉及世界与试炼，遵循“同核异皮”
- 优先抽公共逻辑，不复制粘贴大段代码

涉及文件：
- [文件列表]

验收输出：
- 说明改动点和为什么安全
- 列出潜在风险
- 提供完整回归步骤
```

## 3) Bug 修复模板

```text
请按“复现-定位-最小修复-回归”的流程修复 bug。

问题现象：
- [现象]

复现步骤：
- [步骤]

约束：
- 最小改动，不做无关重构
- 修复后必须跑关联路径回归

输出：
- 根因
- 修复点
- 回归结果
```

## 4) 重构模板

```text
请对以下模块做小步重构，目标是降低耦合并保持行为不变。

模块：
- [模块名]

重构目标：
- [目标]

约束：
- 不改对外行为
- 不改中文文案

```

## 5) 提交前质量检查模板

```text
请作为质量审查员执行提交前检查。

检查范围：
- [文件或目录]

重点检查：
- 节点路径与脚本引用一致性
- 输入映射一致性
- 单机/联机分支风险
- 世界/试炼一致性

输出：
- 高/中/低风险清单
- 修复建议（最小方案）
- 建议回归路径
```
