# Godot UI 页面落地

## 目标

把设计稿/线框图转成可运行的 Godot 页面，并与当前项目结构保持一致。

## 适用范围

- 大厅、登录/注册、世界 HUD、个人中心、各类 Overlay。
- 新页面开发或已有页面改版。

## 执行步骤

1. 先确认目标页面和状态（默认态/错误态/加载态/空态）。
2. 对照现有场景节点，优先复用已有组件和脚本。
3. 先搭结构层级，再做间距和尺寸，最后接信号与交互。
4. 保留中文文案，避免无必要改字。
5. 改完后检查移动端与 PC 的布局一致性。

## 项目对照

- 页面蓝图：`docs/FLOW_UI_DESIGN_BLUEPRINT.md`
- 常用场景：`Scenes/ui/`
- 常用脚本：`Scripts/ui/`、`Scripts/meta/`、`Scripts/world/`

## 输出格式

- 本次改动页面列表（场景 + 脚本）
- 交互说明（触发、反馈、关闭/返回路径）
- 手测清单（PC + 移动端）

## 完成前检查

- [ ] 节点路径未破坏既有脚本引用
- [ ] 按钮/输入框可点击且焦点行为正常
- [ ] Overlay 打开/关闭无冲突
- [ ] 关键分辨率下布局不挤压
