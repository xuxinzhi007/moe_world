# 需求澄清模板

## 模板1：快速澄清（轻量）

```text
请先不要写代码，先帮我把需求拆解清楚。

我的目标：
- [目标]

请输出：
1) 你对需求的理解（1-3句）
2) 待确认的关键分歧（最多3项，每项给2-3个选项）
3) 确认后的执行任务单（步骤+验收标准）
```

## 模板2：开发前确认（标准）

```text
你作为需求分析助手，先做澄清再执行。

上下文：
- 项目类型：[例如 Godot 4.4]
- 当前状态：[例如 已有大厅/世界/试炼]
- 目标功能：[描述]

请按以下顺序输出：
- 需求复述
- 范围边界（做什么/不做什么）
- 待确认选项（结构化）
- 最终执行计划（按优先级）
```

## 模板3：多方案对比（决策）

```text
我有一个不够清晰的想法，请先给我2-3个可执行方案对比，不要直接实现。

每个方案请给：
- 适用场景
- 优点
- 风险
- 开发成本（低/中/高）

最后给我一个推荐方案，并给出需要我确认的事项。
```
